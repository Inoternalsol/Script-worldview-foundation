## 🧭 AGENT MISSION STATEMENT

You are a senior full-stack engineer tasked with building a complete, production-ready, enterprise-grade website platform for **Script Worldview Foundation** — a faith-inspired Nigerian NGO focused on education, humanitarian response, and community development.

This is NOT a simple brochure website. You are building a **full-featured digital platform** with 22 public pages, a role-based admin CMS, donation/payment processing, an AI chatbot, a transactional email system, a media library, and a structured relational database. The platform must rival the online presence of leading NGOs like UNICEF, World Vision, and Oxfam — while remaining distinctly warm, faith-inspired, and values-driven.

Work **phase by phase, file by file, feature by feature** until the entire platform is complete. Never stop mid-phase. Never skip a feature. Confirm each phase is done before moving to the next.

```

CTA/Action: #E65100  (Warm Orange)       — donation buttons, urgent CTAs
Background: #F9FAFB  (Off-white)         — page backgrounds
Text Dark:  #1C2833  (Deep Charcoal)     — body text
Text Muted: #5D6D7E  (Slate Gray)        — captions, labels
Success:    #27AE60  (Green)             — confirmations
Error:      #E74C3C  (Red)               — validation errors
```

### Typography
```
Heading:   Playfair Display (serif)   — trust, authority
Body:      Inter or Nunito Sans       — clean, legible
Quotes:    Lora (serif)               — testimonials, pull quotes
Code:      JetBrains Mono             — technical content
```
Load all fonts via `next/font` for performance.

### Component Design Rules
- Card border-radius: `12px`
- Button height: `48px` minimum (touch target compliance)
- Button variants: Primary (filled navy), Secondary (outlined), Ghost, Danger (red)
- Cards: hover elevation (subtle shadow increase + 2px translateY)
- Navigation: sticky header, transparent→solid on scroll, mobile hamburger drawer
- Hero sections: full-width, video/parallax support, overlay, animated text entry
- Forms: floating labels, real-time Zod validation, clear error states
- Loading states: skeleton screens (not spinners) for content areas
- Toast notifications (Shadcn Toast) for all async actions

## 🗃️ DATABASE SCHEMA — CLOUDFLARE D1

Create all 15 tables with Drizzle ORM migrations. Every table must include `created_at` and `updated_at` timestamps (auto-managed).

```sql
-- 1. users
id, name, email, password_hash, role ENUM('super_admin','dept_admin','content_editor','viewer'),
department, status ENUM('active','suspended'), avatar_url, last_login, created_at, updated_at

-- 2. roles_permissions
id, role_name, permissions_json TEXT, created_by, created_at, updated_at

-- 3. pages
id, slug UNIQUE, title, content_json TEXT, meta_title, meta_desc, og_image,
status ENUM('draft','published'), author_id FK→users, created_at, updated_at

-- 4. blog_posts
id, title, slug UNIQUE, content TEXT, excerpt, category_id, author_id FK→users,
featured_image, status ENUM('draft','published','archived'), published_at,
view_count DEFAULT 0, read_time_minutes, created_at, updated_at

-- 5. programs
id, name, slug UNIQUE, description, category ENUM('education','humanitarian','community','research','capacity'),
icon, featured_image, status ENUM('active','inactive'), sort_order, created_at, updated_at

-- 6. events
id, title, slug, date, end_date, location, address, description, capacity,
registrations_count DEFAULT 0, featured_image, speakers_json TEXT,
status ENUM('upcoming','ongoing','past','cancelled'), created_by FK→users, created_at, updated_at

-- 7. event_registrations
id, event_id FK→events, name, email, phone, organization, role_title,
dietary_needs, accessibility_needs, status ENUM('confirmed','cancelled','waitlist'),
registered_at, created_at, updated_at

-- 8. donations
id, donor_name, donor_email, donor_phone, amount, currency, campaign_id FK→campaigns,
payment_ref UNIQUE, gateway ENUM('paystack','stripe'), status ENUM('pending','completed','failed','refunded'),
anonymous BOOLEAN DEFAULT false, dedication_message, donated_at, created_at, updated_at

-- 9. campaigns
id, title, slug, goal_amount, raised_amount DEFAULT 0, deadline,
description, featured_image, status ENUM('active','completed','paused'),
donor_count DEFAULT 0, created_by FK→users, created_at, updated_at

-- 10. contacts
id, name, email, phone, subject, department ENUM('general','education','humanitarian','community','hr','press','partnership'),
message, type ENUM('contact','partnership','community','press','humanitarian'),
status ENUM('new','in_progress','resolved'), assigned_to FK→users,
replied_at, source, created_at, updated_at

-- 11. volunteers
id, name, email, phone, location, skills_json TEXT, availability_json TEXT,
languages, motivation, how_did_you_hear,
status ENUM('pending','approved','active','rejected'), applied_at, created_at, updated_at

-- 12. newsletter_subscribers
id, email UNIQUE, first_name, last_name, preferences_json TEXT,
status ENUM('active','unsubscribed'), subscribed_at, unsubscribed_at, created_at, updated_at

-- 13. job_postings
id, title, department, location, type ENUM('full_time','part_time','contract','volunteer','internship'),
description, requirements, deadline, status ENUM('open','closed','archived'),
created_by FK→users, created_at, updated_at

-- 14. job_applications
id, job_id FK→job_postings, name, email, phone, years_experience,
cv_url, cover_letter, linkedin_url,
status ENUM('received','reviewing','shortlisted','rejected','hired'),
applied_at, created_at, updated_at

-- 15. media_library
id, filename, url, type ENUM('image','video','document'), size_bytes,
alt_text, category, tags_json TEXT, uploaded_by FK→users, created_at, updated_at

-- 16. audit_logs
id, user_id FK→users, action, resource, resource_id, details_json TEXT,
ip_address, user_agent, timestamp
```

### ═══ PHASE 10: SECURITY IMPLEMENTATION ═══

**10.1 — Authentication**
- bcrypt password hashing (12 rounds) on account creation
- JWT sessions via NextAuth.js — secure, httpOnly cookies
- Session expiry: 8 hours inactivity, 24 hours maximum
- Failed login lockout: 5 attempts → 15-minute lockout (track in D1)
- MFA for Super Admin: TOTP (use `otplib`) or email OTP

**10.2 — API Security**
- Zod schema validation on ALL API inputs (throw 400 on invalid)
- Parameterized queries via Drizzle ORM (prevents SQL injection)
- CSRF protection: validate Origin header on state-changing routes
- Rate limiting: Cloudflare rate limiting rules per IP per endpoint
- IP-based blocking for repeated abuse

**10.3 — Headers**
Configure in `next.config.mjs`:
```javascript
headers: [
  { key: 'X-Frame-Options', value: 'DENY' },
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
  { key: 'Permissions-Policy', value: 'camera=(), microphone=(), geolocation=()' },
  { key: 'Content-Security-Policy', value: "default-src 'self'; ..." },
]
```

**10.4 — Compliance**
- Cookie consent banner (Shadcn Dialog): Essential (required), Analytics (optional), Marketing (optional). Store preference in localStorage.
- Privacy Policy page with NDPR/GDPR-grade content
- Unsubscribe endpoint: `GET /api/newsletter/unsubscribe?token=...` — one-click, no login required
